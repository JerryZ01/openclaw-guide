# ❌ 错误代码详解

本文档列出 OpenClaw 常见错误及其解决方案。

## 错误格式

```json
{
  "error": {
    "code": 1001,
    "message": "错误描述",
    "details": {}
  }
}
```

---

## 认证错误 (1xxx)

### 1001 - 无效 Token

**原因**: Token 错误或过期

**解决**:
```bash
# 重新生成 Token
openssl rand -hex 32

# 更新配置
openclaw config set gateway.auth.token "新token"
```

### 1002 - Token 缺失

**原因**: 请求未携带 Token

**解决**:
```bash
# 添加 Header
curl -H "Authorization: Bearer YOUR_TOKEN" http://localhost:16784/api/status
```

### 1003 - Token 权限不足

**原因**: Token 无权限执行操作

**解决**: 检查 Token 是否配置了足够的权限

## 配置错误 (2xxx)

### 2001 - 配置文件无效

**原因**: JSON 格式错误

**解决**:
```bash
# 验证 JSON
cat config.json | python3 -m json.tool

# 或使用 OpenClaw 检查
openclaw doctor
```

### 2002 - 缺少必需配置

**原因**: 必填字段缺失

**解决**:
```bash
# 查看需要什么配置
openclaw config get --missing
```

### 2003 - API Key 无效

**原因**: API Key 格式错误或过期

**解决**:
```bash
# 验证 Key
curl -H "Authorization: Bearer $ANTHROPIC_API_KEY" \
  https://api.anthropic.com/v1/messages -X POST \
  -H "anthropic-version: 2023-06-01" \
  -d '{"model":"claude-4-haiku-20250514","max_tokens":1,"messages":[{"role":"user","content":"hi"}]}'
```

## 渠道错误 (3xxx)

### 3001 - Telegram 连接失败

**原因**: Bot Token 错误

**解决**:
```bash
# 验证 Token
curl https://api.telegram.org/bot<TOKEN>/getMe

# 重新配置
openclaw config set channels.telegram.botToken "正确token"
```

### 3002 - 飞书 App 验证失败

**原因**: App ID/Secret 错误

**解决**:
1. 检查 [飞书开放平台](https://open.feishu.cn/)
2. 确认 App 已在对应群聊添加
3. 验证配置:
```bash
openclaw config get channels.feishu
```

### 3003 - 企业微信回调失败

**原因**: URL 验证失败

**解决**:
```bash
# 检查 Token
openclaw config get channels.wecom

# 确保回调 URL 可公网访问
ngrok http 16784
```

### 3004 - WhatsApp 连接失败

**原因**: Phone Number ID 或 Access Token 错误

**解决**:
1. 检查 Facebook Developer 后台
2. 确认 Token 未过期
3. 验证 Webhook

### 3005 - Discord 连接失败

**原因**: Bot Token 错误或缺少权限

**解决**:
```bash
# 检查权限
# 需要: READ_MESSAGES, SEND_MESSAGES, EMBED_LINKS
```

## 模型错误 (4xxx)

### 4001 - API 请求失败

**原因**: 网络问题或 API 服务商故障

**解决**:
```bash
# 检查网络
ping api.anthropic.com

# 切换备用模型
openclaw config set model.provider openai
```

### 4002 - 模型配额不足

**原因**: API 账户余额不足

**解决**:
```bash
# 检查配额
curl -H "x-api-key: $ANTHROPIC_API_KEY" \
  https://api.anthropic.com/v1/usage
```

### 4003 - 超出速率限制

**原因**: 请求过快

**解决**:
```json
{
  "rateLimit": {
    "enabled": true,
    "requestsPerMinute": 60
  }
}
```

### 4004 - 上下文过长

**原因**: 消息历史超出模型限制

**解决**:
```json
{
  "memory": {
    "maxMessages": 20
  }
}
```

### 4005 - 内容被拒绝

**原因**: 违反模型安全政策

**解决**: 检查消息内容，移除违规部分

## 技能错误 (5xxx)

### 5001 - 技能加载失败

**原因**: 技能代码错误

**解决**:
```bash
# 查看详细错误
openclaw logs --level debug | grep skill

# 检查技能代码
cat skills/my-skill/index.js
```

### 5002 - 技能未找到

**原因**: 技能不存在

**解决**:
```bash
# 列出可用技能
openclaw skills list

# 安装技能
openclaw skills install skill-name
```

### 5003 - 技能权限不足

**原因**: 技能尝试执行未授权操作

**解决**:
```json
{
  "skills": {
    "my-skill": {
      "permissions": ["read", "write", "message"]
    }
  }
}
```

## 系统错误 (6xxx)

### 6001 - 端口被占用

**原因**: 16784 端口被其他程序占用

**解决**:
```bash
# 查看占用
lsof -i :16784

# 换端口
openclaw serve --port 16785

# 或杀死占用进程
kill $(lsof -t -i :16784)
```

### 6002 - 内存不足

**原因**: 系统内存不足

**解决**:
```bash
# 查看内存
free -h

# 增加 swap
fallocate -l 2G /swapfile
chmod 600 /swapfile
mkswap /swapfile
swapon /swapfile
```

### 6003 - 磁盘空间不足

**原因**: 磁盘满了

**解决**:
```bash
# 查看磁盘
df -h

# 清理日志
rm -rf logs/*.log.*
openclaw logs cleanup

# 清理缓存
rm -rf ~/.openclaw/cache
```

### 6004 - 权限拒绝

**原因**: 文件/目录权限问题

**解决**:
```bash
# 修复权限
chown -R $(whoami) ~/.openclaw
chmod -R 755 ~/.openclaw
```

## 网络错误 (7xxx)

### 7001 - 连接超时

**原因**: 网络延迟或阻塞

**解决**:
```bash
# 测试网络
curl -m 10 https://api.anthropic.com

# 检查防火墙
sudo iptables -L
```

### 7002 - DNS 解析失败

**原因**: DNS 问题

**解决**:
```bash
# 更换 DNS
echo "nameserver 8.8.8.8" | sudo tee /etc/resolv.conf
```

### 7003 - SSL 证书错误

**原因**: 证书验证失败

**解决**:
```bash
# 更新证书
sudo apt update && sudo apt install ca-certificates

# 或忽略验证（仅开发）
NODE_TLS_REJECT_UNAUTHORIZED=0 openclaw serve
```

## 消息错误 (8xxx)

### 8001 - 消息发送失败

**原因**: 渠道问题或目标不存在

**解决**:
```bash
# 检查渠道状态
openclaw channel list

# 验证目标 ID
openclaw status --verbose | grep -A5 telegram
```

### 8002 - 目标用户不存在

**原因**: 用户 ID 错误

**解决**: 确认用户是否在群里/已添加 bot

### 8003 - 消息过长

**原因**: 超出渠道限制

**解决**: 拆分消息或使用摘要

## Cron 错误 (9xxx)

### 9001 - 定时任务执行失败

**原因**: 任务配置错误

**解决**:
```bash
# 查看任务日志
openclaw logs | grep "cron"

# 测试手动执行
openclaw cron run job_1
```

### 9002 - 定时任务时间格式错误

**原因**: Cron 表达式错误

**解决**: 使用标准 cron 格式
```
# 分 时 日 月 周
0 9 * * *     # 每天 9 点
0 9 * * 1-5   # 工作日 9 点
*/15 * * * *  # 每 15 分钟
```
