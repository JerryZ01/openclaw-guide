# ☁️ 系统服务部署

本文档介绍如何将 OpenClaw 配置为系统服务，实现开机自启和后台运行。

## systemd (Linux 推荐)

### 创建服务文件

```bash
mkdir -p ~/.config/systemd/user
```

```ini
# ~/.config/systemd/user/openclaw-gateway.service
[Unit]
Description=OpenClaw Gateway
After=network.target

[Service]
Type=simple
ExecStart=/usr/bin/openclaw gateway --port 16784
Restart=on-failure
RestartSec=10
Environment=NODE_ENV=production

# 日志配置
StandardOutput=append:/tmp/openclaw/stdout.log
StandardError=append:/tmp/openclaw/stderr.log

[Install]
WantedBy=default.target
```

### 管理服务

```bash
# 重新加载配置
systemctl --user daemon-reload

# 启动服务
systemctl --user start openclaw-gateway

# 停止服务
systemctl --user stop openclaw-gateway

# 查看状态
systemctl --user status openclaw-gateway

# 开机自启
systemctl --user enable openclaw-gateway

# 禁用自启
systemctl --user disable openclaw-gateway

# 查看日志
journalctl --user -u openclaw-gateway -f
```

### 完整服务文件（推荐）

```ini
[Unit]
Description=OpenClaw Multi-Channel Gateway
Documentation=https://docs.openclaw.ai
After=network-online.target
Wants=network-online.target

[Service]
Type=simple
User=admin
Group=admin
WorkingDirectory=/home/admin
ExecStartPre=/usr/bin/sleep 5
ExecStart=/usr/bin/openclaw gateway --port 16784 --bind 0.0.0.0
Restart=on-failure
RestartSec=15
TimeoutStartSec=30
TimeoutStopSec=30

# 环境变量
Environment=OPENCLAW_PORT=16784
Environment=NODE_ENV=production
Environment=TZ=Asia/Shanghai

# 资源限制
MemoryMax=2G
CPUQuota=200%

# 日志
SyslogIdentifier=openclaw

[Install]
WantedBy=default.target
```

## pm2 (跨平台)

### 安装 pm2

```bash
npm install -g pm2
```

### 启动服务

```bash
pm2 start openclaw -- gateway --port 16784
```

### 配置 pm2

```json
// ecosystem.config.js
module.exports = {
  apps: [{
    name: 'openclaw',
    script: 'openclaw',
    args: 'gateway --port 16784',
    cwd: '/home/admin',
    instances: 1,
    autorestart: true,
    watch: false,
    max_memory_restart: '2G',
    env: {
      NODE_ENV: 'production',
      TZ: 'Asia/Shanghai'
    },
    error_file: '/tmp/openclaw/err.log',
    out_file: '/tmp/openclaw/out.log',
    log_date_format: 'YYYY-MM-DD HH:mm:ss',
    merge_logs: true
  }]
}
```

### 管理命令

```bash
# 启动
pm2 start ecosystem.config.js

# 重启
pm2 restart openclaw

# 停止
pm2 stop openclaw

# 查看状态
pm2 status

# 查看日志
pm2 logs openclaw

# 开机自启
pm2 startup
pm2 save
```

## Docker systemd (推荐用于容器)

### 创建 Docker 服务

```ini
# /etc/systemd/system/openclaw.service
[Unit]
Description=OpenClaw Docker Container
Requires=docker.service
After=docker.service

[Service]
Type=oneshot
RemainAfterExit=yes
ExecStart=/usr/bin/docker start openclaw
ExecStop=/usr/bin/docker stop openclaw

[Install]
WantedBy=multi-user.target
```

### 管理命令

```bash
# 重新加载
sudo systemctl daemon-reload

# 启动
sudo systemctl start openclaw

# 开机自启
sudo systemctl enable openclaw
```

## 开机自启对比

| 方式 | 优点 | 缺点 |
|------|------|------|
| systemd | Linux 原生，稳定 | 仅限 Linux |
| pm2 | 跨平台，监控丰富 | 需要额外安装 |
| Docker | 环境隔离，易迁移 | 需要 Docker |

## 高级配置

### 多实例负载均衡

```yaml
# docker-compose.yml
version: '3.8'

services:
  openclaw-1:
    image: openclaw/openclaw:latest
    ports:
      - "16784:16784"
    environment:
      - OPENCLAW_INSTANCE=1

  openclaw-2:
    image: openclaw/openclaw:latest
    ports:
      - "16785:16784"
    environment:
      - OPENCLAW_INSTANCE=2

  nginx:
    image: nginx:latest
    ports:
      - "80:80"
    volumes:
      - ./nginx.conf:/etc/nginx/nginx.conf:ro
```

### 健康检查

```bash
# 脚本方式
#!/bin/bash
response=$(curl -s -o /dev/null -w "%{http_code}" http://127.0.0.1:16784/health)
if [ "$response" != "200" ]; then
  echo "Health check failed"
  exit 1
fi
```

### 自动重启 Cron

```bash
# 每天凌晨 3 点重启
0 3 * * * systemctl --user restart openclaw-gateway
```

## 监控

### 查看资源使用

```bash
# systemd
systemctl --user status openclaw-gateway

# pm2
pm2 monit

# Docker
docker stats openclaw
```

### 日志管理

```bash
# journald 日志
journalctl --user -u openclaw-gateway --since "1 hour ago"

# logrotate 配置
# /etc/logrotate.d/openclaw
/tmp/openclaw/*.log {
  daily
  rotate 7
  compress
  delaycompress
  notifempty
  create 0640 admin admin
}
```

## 故障排查

### 服务启动失败

```bash
# 检查状态
systemctl --user status openclaw-gateway

# 查看日志
journalctl --user -xe

# 手动运行测试
openclaw gateway --port 16784
```

### 端口被占用

```bash
# 查找占用
lsof -i :16784

# 杀掉进程
kill -9 <PID>
```

### 内存不足

```bash
# 检查内存
free -h

# 增加 swap
sudo fallocate -l 2G /swapfile
sudo chmod 600 /swapfile
sudo mkswap /swapfile
sudo swapon /swapfile
```

## 生产环境建议

1. **使用 systemd**：Linux 生产环境推荐
2. **配置日志轮转**：避免磁盘爆满
3. **设置资源限制**：防止内存泄漏
4. **监控健康状态**：使用 healthcheck
5. **定期备份配置**：防止数据丢失
