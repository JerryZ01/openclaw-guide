# 🏠 本地开发指南

本文档介绍如何本地开发和测试 OpenClaw。

## 环境要求

- Node.js 22+
- npm 10+
- Git

## 快速开始

### 1. 克隆源码

```bash
git clone https://github.com/openclaw/openclaw.git
cd openclaw
```

### 2. 安装依赖

```bash
npm install
```

### 3. 开发模式

```bash
# 启动开发服务器
npm run dev

# 运行测试
npm test

# 代码检查
npm run lint
```

## 项目结构

```
openclaw/
├── src/
│   ├── index.ts          # 入口
│   ├── gateway/          # Gateway 核心
│   ├── channels/         # 渠道实现
│   ├── agents/           # Agent 处理
│   ├── skills/           # 技能系统
│   └── utils/            # 工具函数
├── dist/                 # 编译输出
├── test/                 # 测试文件
└── docs/                 # 项目文档
```

## 开发指南

### 添加新渠道

1. 在 `src/channels/` 创建新目录
2. 实现 `Channel` 接口
3. 注册到 `ChannelManager`

示例：

```typescript
// src/channels/mychannel/index.ts
import { Channel, Message, ChannelOptions } from '../interface';

export class MyChannel implements Channel {
  name = 'mychannel';
  
  async sendMessage(target: string, message: Message): Promise<void> {
    // 实现发送逻辑
  }
  
  async start(options: ChannelOptions): Promise<void> {
    // 启动连接
  }
  
  async stop(): Promise<void> {
    // 停止连接
  }
}
```

### 添加新技能

```typescript
// src/skills/my-skill/index.ts
export default {
  name: 'my-skill',
  version: '1.0.0',
  
  rules: [
    {
      pattern: /hello/,
      response: '你好！'
    }
  ],
  
  async onLoad() {
    console.log('技能加载');
  }
};
```

## 代码规范

### Git 提交规范

使用 [Conventional Commits](https://www.conventionalcommits.org/)：

```
feat: 添加新功能
fix: 修复问题
docs: 文档更新
style: 格式调整
refactor: 代码重构
test: 测试相关
chore: 构建/工具
```

### ESLint 规则

项目使用 ESLint + Prettier，请确保提交前运行：

```bash
npm run lint
npm run format
```

## 发布

```bash
# 版本更新
npm version major  # 1.0.0 -> 2.0.0
npm version minor  # 1.0.0 -> 1.1.0
npm version patch  # 1.0.0 -> 1.0.1

# 发布到 npm
npm publish
```

## 常见问题

### Q: 编译失败？

```bash
# 清理缓存
rm -rf dist node_modules
npm install
npm run build
```

### Q: 测试失败？

```bash
# 查看详细错误
npm run test:verbose

# 只运行失败的测试
npm run test:only-failed
```
