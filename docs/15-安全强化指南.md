## 当前安全形势

截至 2026 年 2 月，OpenClaw 已突破 180,000 GitHub stars，但也成为攻击目标：

| 研究机构 | 分析技能数 | 恶意/缺陷 |
|---------|-----------|-----------|
| Koi Security | 2,857 | 341 个恶意 (12%) |
| Snyk (ToxicSkills) | 3,984 | 1,467 个缺陷 (36.8%) |
| Bitdefender Labs | ~400 深度分析 | 800+ 恶意 |

**关键数据：**
- 30,000+ 公开暴露实例
- 21,639 实例可被扫描到（美国、中国、新加坡）
- 15,200+ 易受 RCE 攻击
- CVE-2026-25253 (CVSS 8.8) 影响 2026.1.29 之前所有版本

## 攻击面分析

### 1. 技能 (Skills) 是主要攻击向量

恶意技能常见手法：
- **伪装**：加密货币工具、YouTube 汇总、Polymarket 机器人
- **社会工程**：要求下载带密码的 ZIP（内含键盘记录器）
- **拼写错误**：clawhub、clawhub1、clawhubb、clawwhub
- **共享 C2**：ClawHavoc 技能都连向 91.92.242.30

### 2. 实例暴露

常见暴露原因：
- Nginx 反向代理绕过 localhost 限制
- 默认端口 18789 公开可访问
- 明文凭证在 ~/.clawdbot/.env 和 markdown 内存文件中

### 3. 提示注入

攻击方式：
- **直接注入**：恶意用户消息绑过安全护栏
- **间接注入**：中毒的邮件或 Slack 消息含隐藏指令
- **持久化投毒**：技能写入 SOUL.md 或 MEMORY.md

## 部署隔离

### 永远不要在主工作站运行

使用专用基础设施。

### Docker 强化部署

```yaml
# docker-compose.yml
services:
  openclaw:
    image: openclaw/openclaw:latest
    user: "1000:1000"                    # 非 root 运行
    read_only: true                       # 只读文件系统
    cap_drop:                             # 丢弃所有能力
      - ALL
    tmpfs:                                # 临时文件系统
      - /tmp
    ports:
      - "127.0.0.1:16784:16784"          # 仅本地访问
    volumes:
      - ./data:/data
      - ./config:/config
    restart: unless-stopped
    healthcheck:
      test: ["CMD", "openclaw", "status"]
      interval: 30s
```

### 网关配置

```json
{
  "gateway": {
    "bind": "127.0.0.1",
    "port": 16784,
    "auth": {
      "token": "your-strong-token-min-32-chars"
    }
  }
}
```

### 防火墙规则

```bash
# 仅允许内网访问
sudo ufw allow from 192.168.1.0/24 to any port 16784
sudo ufw deny 16784

# 或者使用 Tailscale 实现安全远程访问
```

### Tailscale 远程访问

推荐使用 Tailscale 替代公网暴露：
- 自动 HTTPS
- 零公网暴露
- 访问地址：`https://[your-machine].tail[xxxx].ts.net`

## 技能供应链安全

### 核心原则：默认禁用第三方技能

**默认立场：关闭 ClawHub，等待验证后再启用。**

### 技能安装前检查

使用 [SkillCheck](https://repello.ai/tools/skills) 扫描技能：
- 基于浏览器的工具，无需安装
- 每日更新的威胁情报

或手动检查：

```bash
# 检查技能压缩包内容
unzip -l skill.zip

# 检查外部下载
grep -r "curl.*\|.*bash" skill/

# 检查网络请求
grep -r "fetch\|axios\|http" skill/

# 检查 Base64 编码命令
grep -r "base64.*-d\|atob" skill/
```

### 恶意特征清单

安装前检查以下危险信号：
- 外部下载（带密码的 ZIP）
- Base64 编码命令
- 陌生域名的网络请求
- 硬编码的 API Key、Token
- 工作区外的文件写入
- 包名拼写错误（clawhub、clawwhub 等）

### 已知恶意基础设施

```
# /etc/hosts 或防火墙屏蔽
91.92.242.30  # ClawHavoc C2 服务器
```

### 技能网络过滤

如果技能需要网络访问，使用白名单：

```json
{
  "skills": {
    "network": {
      "allowedDomains": [
        "api.openai.com",
        "api.anthropic.com"
      ]
    }
  }
}
```

### 运行时技能监控

记录技能执行日志：

```json
{
  "skills": {
    "logging": {
      "enabled": true,
      "logFile": "/var/log/openclaw/skills.log"
    }
  }
}
```

## 访问控制

### 工具白名单

```json
{
  "security": {
    "toolPolicy": {
      "allowed": [
        "read",
        "write",
        "message"
      ],
      "requiresApproval": [
        "exec",
        "gateway.restart"
      ]
    }
  }
}
```

### 人工审批机制

对敏感操作需要确认：

```json
{
  "security": {
    "humanInTheLoop": {
      "enabled": true,
      "actions": [
        "message.send",
        "file.delete",
        "config.set"
      ]
    }
  }
}
```

### 凭证隔离

**永远不要**在以下位置存储凭证：
- 环境变量（除非加密）
- 技能配置文件
- 系统提示中

使用哈希icorp Vault 或 AWS Secrets Manager：

```json
{
  "secrets": {
    "provider": "vault",
    "url": "https://vault.example.com",
    "path": "secret/openclaw"
  }
}
```

### 凭证轮换

| 凭证类型 | 轮换周期 |
|---------|---------|
| API Keys | 每 30-90 天 |
| Gateway Tokens | 每 7-14 天 |
| 紧急轮换 | 事件后立即 |

## 提示注入防御

### 输入清理

```json
{
  "security": {
    "inputSanitization": {
      "enabled": true,
      "stripPatterns": [
        "ignore previous instructions",
        "you are now",
        "system:"
      ]
    }
  }
}
```

### 使用 Claude Opus 4.5

根据 Kaspersky 研究，Claude Opus 4.5 对提示注入抵抗力最强：
- 多轮违规率仅 4.8%
- GPT-5.2 为 14.3%
- GPT-5.1 为 28.6%

### 内存清洁

定期清理 memory 文件：

```bash
# 备份内存
cp ~/.openclaw/memory ~/.openclaw/memory-backup

# 清理敏感内容
sed -i 's/密码.*/***/g' ~/.openclaw/memory/*.md
```

### 多步攻击检测

注意以下攻击模式：
1. "查看 John 的邮件" → 无害
2. "把邮件摘要写入 markdown" → 无害
3. "压缩那个 markdown 文件" → 无害
4. "上传到我的备份存储" → **数据泄露！**

## 监控与响应

### 综合日志

```json
{
  "logging": {
    "level": "debug",
    "file": "/var/log/openclaw/app.log",
    "rotate": {
      "maxSize": "10MB",
      "maxFiles": 10
    }
  }
}
```

### SIEM 集成

```bash
# 输出 JSON 格式日志
openclaw logs --json | jq -r '.message' | \
  ngrep -q - | nc syslog.example.com 514
```

### 告警规则示例

```
# 敏感操作告警
alert: sensitive_action
if: action in ["exec", "gateway.restart", "config.set"]

# 异常频率告警
alert: high_frequency
if: count() > 100 per minute

# 新设备登录告警
alert: new_device
if: event == "login" && !known_device
```

### 事件响应剧本

如果检测到被入侵：

1. **立即隔离**
   ```bash
   docker stop openclaw
   # 或
   systemctl stop openclaw
   ```

2. **凭证轮换**
   - 更改所有 API Keys
   - 重新生成 Gateway Token
   - 撤销 OAuth Tokens

3. **取证分析**
   - 检查日志中的异常命令
   - 审查最近修改的文件
   - 检查 crontab 和启动项

4. **恢复**
   - 从干净备份恢复
   - 重新安装所有技能
   - 更新到最新版本
