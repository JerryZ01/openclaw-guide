# ⚙️ OpenClaw 配置详解

## 配置文件位置

- **用户配置**: `~/.openclaw/openclaw.json`
- **日志文件**: `/tmp/openclaw/openclaw-YYYY-MM-DD.log`

## 配置结构

```json
{
  "meta": {
    "lastTouchedVersion": "2026.2.9"
  },
  "gateway": {
    "port": 16784,
    "bind": "0.0.0.0",
    "auth": {
      "token": "your-auth-token"
    }
  },
  "channels": { ... },
  "agents": { ... },
  "models": { ... }
}
```

## Gateway 配置

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `gateway.port` | number | 16784 | Gateway 监听端口 |
| `gateway.bind` | string | "0.0.0.0" | 绑定地址 |
| `gateway.auth.token` | string | 自动生成 | 认证 Token |
| `gateway.remote.token` | string | - | 远程访问 Token |

```json
{
  "gateway": {
    "port": 16784,
    "bind": "lan",
    "auth": {
      "token": "your-secret-token"
    }
  }
}
```

## 使用 CLI 管理配置

```bash
# 查看配置
openclaw config get
openclaw config get gateway.port

# 设置配置
openclaw config set gateway.port 18789
openclaw config set channels.telegram.enabled true

# 删除配置
openclaw config unset channels.telegram.botToken

# 导出配置
openclaw config get > config-backup.json

# 导入配置
openclaw config apply config-backup.json
```

## 环境变量

| 变量 | 说明 |
|------|------|
| `OPENCLAW_PORT` | Gateway 端口 |
| `OPENCLAW_CONFIG_PATH` | 配置文件路径 |
| `OPENCLAW_STATE_DIR` | 状态目录 |
| `OPENCLAW_AUTH_TOKEN` | 认证 Token |

## 配置热重载

修改配置后，无需重启 Gateway：

```bash
# 发送 SIGHUP 信号
kill -HUP $(pgrep -f "openclaw gateway")

# 或使用 CLI
openclaw gateway restart
```
