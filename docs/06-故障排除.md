# 🔧 故障排除指南

本文档帮助您解决 OpenClaw 使用过程中遇到的常见问题。

## 快速诊断

### 一键诊断

```bash
# 运行健康检查
openclaw doctor
```

这将自动检测并尝试修复常见问题。

### 查看日志

```bash
# 查看最近日志
openclaw logs

# 实时查看
openclaw logs --follow

# 查看特定级别
openclaw logs --level error

# 搜索特定关键词
openclaw logs --follow | grep whatsapp
```

## 常见问题

### Gateway 无法启动

#### 问题：端口被占用

```
Error: Port 16784 is already in use
```

**解决方案：**

```bash
# 方法 1：查找占用进程
lsof -i :16784
kill -9 <PID>

# 方法 2：使用其他端口
openclaw gateway --port 18789

# 方法 3：强制重启
openclaw gateway --force
```

#### 问题：配置文件错误

```
Error: Config parse failed
```

**解决方案：**

```bash
# 验证配置文件
openclaw config get

# 重置为默认配置
openclaw reset --config

# 重新初始化
openclaw onboard
```

### 渠道连接问题

#### WhatsApp 登录失败

**症状：** QR 码无法扫描或登录后很快断开

**解决方案：**

```bash
# 1. 清除会话
openclaw channels logout whatsapp

# 2. 重新登录
openclaw channels login whatsapp

# 3. 检查网络
curl -I https://web.whatsapp.com
```

**常见原因：**
- IP 被 WhatsApp 限制（使用代理/VPN）
- 账号异常（需要手机验证）
- 多设备登录限制

#### Telegram 无响应

**症状：** 发送消息后机器人不回复

**解决方案：**

```bash
# 1. 验证 Bot Token
openclaw config get channels.telegram.botToken

# 2. 检查 Bot 设置
# 确保机器人隐私模式已关闭（@BotFather -> /setprivacy）

# 3. 重新设置 webhook
openclaw channels telegram webhook --force

# 4. 检查日志
openclaw logs --follow | grep telegram
```

#### 飞书消息接收不到

**症状：** 发送消息给机器人没有响应

**解决方案：**

```bash
# 1. 验证应用凭证
openclaw config get channels.feishu.appId

# 2. 确认应用已发布
# 登录飞书开放平台检查应用状态

# 3. 检查回调 URL
# 确保公网可访问

# 4. 验证 Token
openclaw config get channels.feishu.verificationToken
```

### 消息发送失败

#### 通用排查步骤

```bash
# 1. 检查渠道状态
openclaw status

# 2. 查看详细日志
openclaw logs --level debug | grep "send message"

# 3. 测试发送
openclaw message send --channel <channel> --target <target> --message "test"
```

#### 常见错误

**错误 1：Unauthorized**

```
Error: unauthorized
```

**原因：** 认证 Token 不匹配

**解决：**

```bash
# 方案 1：重启 Gateway
openclaw gateway restart

# 方案 2：重新配置 Token
openclaw config unset gateway.remote.token
openclaw gateway restart
```

**错误 2：Rate Limited**

```
Error: rate limit exceeded
```

**原因：** 发送频率过高

**解决：**
- 等待一段时间
- 在配置中降低发送频率限制
- 使用消息队列缓冲

### 模型调用问题

#### API Key 无效

**症状：** 模型不响应或返回错误

**解决方案：**

```bash
# 检查配置
openclaw config get models

# 验证 API Key
openclaw models test

# 重新配置
openclaw config set providers.anthropic.apiKey "sk-..."
```

#### 上下文超限

**症状：** 回答被截断或报错

**解决：**
- 减少对话历史长度
- 分段处理长文本
- 增加 maxTokens 配置

### 性能问题

#### Gateway 响应慢

**诊断：**

```bash
# 检查系统资源
top -bn1 | head -20

# 检查内存使用
free -h

# 检查磁盘 IO
iostat -x 1 5
```

**优化建议：**

1. 增加内存
2. 使用 SSD
3. 关闭不需要的渠道
4. 减少日志级别

#### 内存泄漏

**症状：** 长时间运行后内存持续增长

**解决：**

```bash
# 定期重启 Gateway
openclaw gateway restart

# 设置内存限制
openclaw gateway --max-old-space-size 2048
```

### 网络问题

#### 无法连接外网

**症状：** 渠道无法连接

```bash
# 测试网络
ping 8.8.8.8

# 测试 DNS
nslookup api.anthropic.com

# 检查代理
echo $HTTP_PROXY
echo $HTTPS_PROXY
```

#### Webhook 无法访问

**症状：** 回调消息收不到

**解决：**

```bash
# 1. 使用内网穿透
ngrok http 16784

# 2. 或配置公网 IP
openclaw config set gateway.bind "0.0.0.0"

# 3. 检查防火墙
sudo iptables -L -n
```

## 数据恢复

### 恢复配置

```bash
# 导出当前配置
openclaw config get > backup.json

# 从备份恢复
openclaw config apply backup.json
```

### 恢复会话

```bash
# 列出会话
openclaw sessions list

# 恢复会话
openclaw sessions restore <session-id>
```
