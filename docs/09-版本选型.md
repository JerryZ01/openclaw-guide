# 📊 版本对比与选型指南

本文档帮助你在不同场景下选择合适的 OpenClaw 部署方式。

## 部署方式对比

### 1. 直接运行

```bash
# 安装
npm install -g openclaw@latest

# 启动
openclaw gateway --port 16784
```

| 优点 | 缺点 |
|------|------|
| 简单快速 | 需要手动管理进程 |
| 资源占用低 | 更新需要手动 |
| 配置灵活 | 不适合生产环境 |

**适用场景：**
- ✅ 个人使用
- ✅ 开发测试
- ✅ 临时场景

---

### 2. Systemd 服务

```bash
# 创建服务
sudo tee ~/.config/systemd/user/openclaw.service << EOF
[Unit]
Description=OpenClaw Gateway
After=network.target

[Service]
Type=simple
ExecStart=/usr/bin/openclaw gateway --port 16784
Restart=on-failure

[Install]
WantedBy=default.target
EOF

# 启用
systemctl --user enable openclaw
systemctl --user start openclaw
```

| 优点 | 缺点 |
|------|------|
|开机自启 | 仅 Linux |
| 进程管理 | 需要 SSH |
| 日志集成 | |

**适用场景：**
- ✅ 个人 VPS
- ✅ 小团队服务器
- ✅ 长期运行

---

### 3. Docker 部署

```yaml
# docker-compose.yml
version: '3.8'
services:
  openclaw:
    image: openclaw/openclaw:latest
    ports:
      - "16784:16784"
    volumes:
      - ./data:/home/admin/.openclaw
    restart: unless-stopped
```

| 优点 | 缺点 |
|------|------|
| 环境隔离 | 资源额外占用 |
| 快速部署 | 需要 Docker |
| 易于迁移 | 网络配置 |
| 版本管理 | |

**适用场景：**
- ✅ 生产环境
- ✅ 多实例部署
- ✅ 混合部署

---

### 4. Kubernetes

```yaml
# k8s-deployment.yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: openclaw
spec:
  replicas: 2
  selector:
    matchLabels:
      app: openclaw
  template:
    spec:
      containers:
      - name: openclaw
        image: openclaw/openclaw:latest
        ports:
        - containerPort: 16784
        env:
        - name: OPENCLAW_PORT
          value: "16784"
```

| 优点 | 缺点 |
|------|------|
| 高可用 | 复杂度高 |
| 自动伸缩 | 需要 K8s |
| 滚动更新 | 成本较高 |

**适用场景：**
- ✅ 大规模部署
- ✅ 企业级应用
- ✅ 需要高可用

```json
{
  "gateway": { "port": 16784 },
  "channels": {
    "telegram": { "enabled": true }
  },
  "agents": {
    "defaults": {
      "model": { "primary": "alibaba-cloud/MiniMax-M2.5" }
    }
  }
}
```

```json
{
  "gateway": { 
    "port": 16784,
    "bind": "0.0.0.0",
    "auth": { "token": "xxx" }
  },
  "channels": {
    "telegram": { "enabled": true },
    "feishu": { "enabled": true }
  },
  "agents": {
    "defaults": {
      "model": { "primary": "anthropic/claude-3.5-sonnet" },
      "maxTokens": 8192
    }
  },
  "security": {
    "allowList": { "telegram": ["123456"] }
  }
}
```

### 生产配置（企业）

```json
{
  "gateway": { 
    "port": 16784,

    "telegram": { "enabled": true }
```

## 性能优化

### 资源需求

| 规模 | CPU | 内存 | 磁盘 |
|------|-----|------|------|
| 个人 | 1核 | 1GB | 5GB |
| 小团队 | 2核 | 2GB | 10GB |
| 企业 | 4核+ | 4GB+ | 20GB+ |

### 优化建议

1. **减少日志级别** - 生产用 `info`
2. **限制历史长度** - 节省内存
3. **使用缓存** - 加速响应
4. **定期清理** - 保持性能
