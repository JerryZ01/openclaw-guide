# 🔧 问题排查速查表

本文档帮助你快速定位和解决常见问题。

## 按症状分类

### 🚫 无法启动

| 症状 | 原因 | 解决方案 |
|------|------|----------|
| 端口被占用 | 16784 已被占用 | `lsof -i :16784` 查找并关闭 |
| 配置文件错误 | JSON 格式错误 | `openclaw config get` 检查 |
| 权限不足 | 端口 < 1024 | 使用 > 1024 端口或 sudo |
| Node 版本低 | < Node 22 | 升级 Node.js |

```bash
# 排查命令
lsof -i :16784                    # 查占用
cat ~/.openclaw/openclaw.json | jq # 验证配置
node --version                    # Node 版本
```

### 📡 渠道问题

#### WhatsApp

| 症状 | 解决方案 |
|------|----------|
| 无法扫码 | 检查网络，尝试代理 |
| 频繁断开 | 检查账号风控 |
| 消息发不出 | 确认对方未拉黑 |

```bash
# 重置登录
openclaw channels logout whatsapp
openclaw channels login whatsapp
```

#### Telegram

| 症状 | 解决方案 |
|------|----------|
| 机器人不响应 | 检查 Bot Token |
| 隐私模式 | @BotFather 设置 /setprivacy |
| 无法入群 | 设为管理员 |

```bash
# 检查配置
openclaw config get channels.telegram
```

#### 飞书

| 症状 | 解决方案 |
|------|----------|
| 收不到消息 | 检查回调 URL |
| 发送失败 | 验证 App ID/Secret |
| 权限不足 | 检查应用权限 |

### 🤖 Agent 问题

| 症状 | 原因 | 解决方案 |
|------|------|----------|
| 不回复 | API Key 错误 | 检查配置 |
| 回答很慢 | 网络或模型 | 检查网络 |
| 回答很短 | maxTokens 小 | 调大 |
| 回答乱码 | 编码问题 | 检查日志 |

```bash
# 测试模型
openclaw models test

# 查看模型配置
openclaw config get agents.defaults.model
```

## 错误代码

### 常见 HTTP 状态码

| 码 | 含义 | 解决 |
|---|------|------|
| 400 | 请求错误 | 检查参数 |
| 401 | 未授权 | 检查 Token |
| 403 | 禁止 | 检查权限 |
| 404 | 未找到 | 检查路径 |
| 429 | 限流 | 稍后重试 |
| 500 | 服务端错误 | 查看日志 |

### 常见 CLI 错误

```
Error: Port already in use
  → 端口被占用，换一个或杀掉进程

Error: Config parse failed
  → JSON 格式错误，检查配置文件

Error: unauthorized
  → Token 不匹配，重启 Gateway

Error: channel not enabled
  → 渠道未启用

Error: session not found
  → 会话过期或不存在
```

## 调试技巧

### 1. 开启调试日志

```bash
openclaw gateway --log-level debug
```

### 2. 过滤特定渠道

```bash
openclaw logs --follow | grep telegram
```

### 3. 检查进程

```bash
ps aux | grep openclaw
```

### 4. 网络诊断

```bash
# 检查端口
nc -zv localhost 16784

# 检查路由
traceroute api.anthropic.com
```

### 5. 一键诊断

```bash
openclaw doctor
```

## 急救命令

```bash
# 1. 完全重启
openclaw gateway stop
openclaw gateway start

# 2. 重置配置
openclaw reset --config
openclaw onboard

# 3. 清理日志
rm -f /tmp/openclaw/openclaw-*.log

# 4. 重新安装
npm install -g openclaw@latest

# 5. 查看状态
openclaw status
openclaw logs
```
