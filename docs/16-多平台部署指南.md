# 📱 多平台部署详细指南

## 概述

本文档详细介绍 OpenClaw 在各种硬件平台和环境下的部署方案，包括树莓派、Mac Mini、Windows WSL2、云 VPS 等。

## 平台对比

| 平台 | 最低配置 | 推荐配置 | 适合场景 |
|-----|---------|---------|---------|
| 树莓派 4/5 | 4GB RAM | 8GB RAM | 轻量使用、学习 |
| Mac Mini | M1 | M2+ | 家庭助手、主力 |
| VPS (云) | 2vCPU/4GB | 4vCPU/8GB | 7×24 运行 |
| NAS | 因特尔 N100+ | i3+ 8GB | 已有 NAS 用户 |
| WSL2 | WSL2 + 4GB | WSL2 + 8GB | Windows 开发 |

## 树莓派部署

### 硬件推荐

- **树莓派 5** 8GB - 最佳性价比
- **树莓派 4** 8GB - 够用
- 存储：32GB+ microSD 卡或 NVMe SSD

### 系统安装

```bash
# 1. 下载 Raspberry Pi OS (64-bit)
# https://www.raspberrypi.com/software/operating-systems/

# 2. 格式化并烧录
sudo dd if=2026-02-12-raspios-bookworm-arm64.img of=/dev/sdX bs=4M status=progress

# 3. 开启 SSH 和 WiFi（headless 模式）
# 在 boot 分区创建空文件 ssh
touch /boot/firmware/ssh

# 创建 wpa_supplicant.conf
cat > /boot/firmware/wpa_supplicant.conf <<EOF
country=CN
network={
    ssid="YourWiFi"
    psk="YourPassword"
}
EOF
```

### Node.js 安装

```bash
# 更新系统
sudo apt update && sudo apt upgrade -y

# 安装 Node.js 22
curl -fsSL https://deb.nodesource.com/setup_22.x | sudo -E bash -
sudo apt install -y nodejs

# 验证
node -v  # 应显示 v22.x.x
```

### OpenClaw 安装

```bash
# 安装 OpenClaw
curl -fsSL https://openclaw.ai/install.sh | bash

# 或者使用 npm
sudo npm install -g openclaw

# 启动配置
openclaw onboarding
```

### 性能优化

```bash
# 启用 zram 增加可用内存
sudo apt install -y zram-tools

# 配置 /etc/default/zramswap
PERCENT=50

# 重启服务
sudo systemctl restart zramswap
```

### 散热处理

```bash
# 安装散热片和风扇
# 添加温度监控
watch -n 1 vcgencmd measure_temp
```

## Mac Mini 部署

### 硬件推荐

- **Mac Mini M1** - 性价比之选
- **Mac Mini M2/M4** - 性能更强
- 存储：256GB+ SSD
- 内存：16GB 更佳

### 安装步骤

```bash
# 使用 Homebrew 安装
brew install openclaw

# 或使用 installer script
curl -fsSL https://openclaw.ai/install.sh | bash

# 启动配置
openclaw onboarding
```

### 启动项配置

```bash
# 创建 LaunchAgent (用户级)
mkdir -p ~/Library/LaunchAgents
cat > ~/Library/LaunchAgents/com.openclaw.plist <<EOF
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
    <key>Label</key>
    <string>com.openclaw</string>
    <key>ProgramArguments</key>
    <array>
        <string>/usr/local/bin/openclaw</string>
        <string>serve</string>
    </array>
    <key>RunAtLoad</key>
    <true/>
    <key>KeepAlive</key>
    <true/>
</dict>
</plist>
EOF

# 启动服务
launchctl load ~/Library/LaunchAgents/com.openclaw.plist
```

### 网络配置

```bash
# macOS 防火墙
sudo /usr/libexec/ApplicationFirewall/socketfilterfw --add /usr/local/bin/openclaw
sudo /usr/libexec/ApplicationFirewall/socketfilterfw --unblockapp /usr/local/bin/openclaw
```

## Windows WSL2 部署

### WSL2 安装

```powershell
# 以管理员身份打开 PowerShell
wsl --install

# 重启后自动安装 Ubuntu

# 或者指定版本
wsl --install -d Ubuntu-22.04
```

### Ubuntu 配置

```bash
# 更新
sudo apt update && sudo apt upgrade -y

# 安装 Node.js 22
curl -fsSL https://deb.nodesource.com/setup_22.x | sudo -E bash -
sudo apt install -y nodejs

# 安装 Docker (可选)
curl -fsSL https://get.docker.com | sh
sudo usermod -aG docker $USER
```

### OpenClaw 安装

```bash
# 安装 OpenClaw
curl -fsSL https://openclaw.ai/install.sh | bash

# 配置
openclaw onboarding

# 从 Windows 访问
# 浏览器打开 http://localhost:16784
```

```bash

cd /mnt/c/Users/YourName/

# 在 Windows 中访问 WSL 文件
# 运行 \\wsl$\Ubuntu\home\username\
```

### Hetzner 部署示例

```bash
# 1. 创建服务器 (Cloud Console 或 CLI)
hcloud server create --name openclaw --image ubuntu-22.04 --location fsn1 --type cpx31

# 2. SSH 登录
ssh root@your-server-ip

# 3. 安装 Docker
curl -fsSL https://get.docker.com | sh

# 4. 创建目录
mkdir -p openclaw/{config,data,logs}

# 5. 创建 docker-compose.yml
cat > openclaw/docker-compose.yml <<EOF
version: '3.8'
services:
  openclaw:
    image: openclaw/openclaw:latest
    container_name: openclaw
    restart: unless-stopped
    ports:
      - "127.0.0.1:16784:16784"
    environment:
      - OPENCLAW_CONFIG=/config/openclaw.json
    volumes:
      - ./config:/config
      - ./data:/data
      - ./logs:/logs
    user: "1000:1000"
    healthcheck:
      test: ["CMD", "openclaw", "status"]
      interval: 30s
      timeout: 10s
      retries: 3

  watchtower:
    image: containrrr/watchtower
    container_name: watchtower
    restart: unless-stopped
    environment:
      - WATCHTOWER_CLEANUP=true
      - WATCHTOWER_INCLUDE_STOPPED=true
    volumes:
      - /var/run/docker.sock:/var/run/docker.sock
    command: --interval 3600 openclaw
EOF

# 6. 启动
cd openclaw && docker compose up -d
```

### 安全的远程访问

#### 方式 1: Tailscale (推荐)

```bash
# 安装 Tailscale
curl -fsSL https://tailscale.com/install.sh | sh

# 登录
tailscale up

# 获取访问地址
tailscale status
# 访问: https://[hostname].tail[xxxx].ts.net:16784
```

#### 方式 2: Cloudflare Tunnel

```bash
# 安装 cloudflared
curl -sSL https://github.com/cloudflare/cloudflared/releases/latest/download/cloudflared-linux-amd64 -o /usr/local/bin/cloudflared
chmod +x /usr/local/bin/cloudflared

# 登录并创建隧道
cloudflared tunnel login
cloudflared tunnel create openclaw

# 配置
cat > /etc/cloudflared/config.yml <<EOF
tunnel: openclaw
credentials-file: /root/.cloudflared/credentials.json
ingress:
  - hostname: openclaw.yourdomain.com
    service: http://localhost:16784
  - service: http_status:404
EOF

# 启动
cloudflared tunnel route dns openclaw openclaw.yourdomain.com
systemctl enable --now cloudflared
```

## Docker Compose 生产配置

### 完整配置示例

```yaml
version: '3.8'

services:
  openclaw:
    image: openclaw/openclaw:latest
    container_name: openclaw
    restart: unless-stopped
    ports:
      - "127.0.0.1:16784:16784"
    environment:
      - NODE_ENV=production
    env_file:
      - .env
    volumes:
      - ./config:/config
      - ./data:/data
      - ./logs:/logs
    user: "1000:1000"
    read_only: true
    security_opt:
      - no-new-privileges:true
    cap_drop:
      - ALL
    networks:
      - openclaw-net
    healthcheck:
      test: ["CMD", "openclaw", "status", "--json"]
      interval: 30s
      timeout: 10s
      retries: 3
      start_period: 60s

  watchtower:
    image: containrrr/watchtower
    container_name: watchtower
    restart: unless-stopped
    environment:
      - WATCHTOWER_CLEANUP=true
      - WATCHTOWER_INCLUDE_STOPPED=true
      - WATCHTOWER_NOTIFICATIONS=shoutrrr
      - WATCHTOWER_NOTIFICATION_URL=discord://webhook
    volumes:
      - /var/run/docker.sock:/var/run/docker.sock
    command: --interval 3600 --label-enable openclaw
    depends_on:
      - openclaw

networks:
  openclaw-net:
    driver: bridge
```

### .env 文件

```bash
# API 配置
```

## 硬件性能对比

### 推理速度 (本地模型)

| 设备 | 模型 | Tokens/sec |
|-----|------|-----------|
| 树莓派 5 8GB | Qwen2-0.5B | 15-20 |
| Mac Mini M1 | Qwen2-1.5B | 30-40 |
| Mac Mini M2 | Qwen2-3B | 45-60 |
| Hetzner CPX31 | Qwen2-1.5B | 25-35 |

### 建议

- **轻量使用**（消息汇总、提醒）: 树莓派 5
- **中度使用**（日程、邮件）: Mac Mini M1 或 VPS
- **重度使用**（开发、多代理）: Mac Mini M2+ 或高配 VPS
