# ⏰ 定时任务 (Cron)

OpenClaw 内置强大的定时任务系统，支持一次性提醒和周期性任务。

## 基本概念

### 任务类型

| 类型 | 说明 | 适用场景 |
|------|------|----------|
| `at` | 一次性任务 | 提醒、会议通知 |
| `every` | 循环间隔任务 | 定期检查、轮询 |
| `cron` | Cron 表达式任务 | 定时执行、周期任务 |

### 会话目标

| 目标 | 说明 |
|------|------|
| `main` | 主会话（需要 systemEvent） |
| `isolated` | 隔离会话（独立运行） |

## 使用方法

### CLI 命令

```bash
# 列出所有任务
openclaw cron list

# 手动触发任务
openclaw cron run <jobId>

# 删除任务
openclaw cron remove <jobId>

# 查看运行历史
openclaw cron runs <jobId>
```

### 一次性任务 (at)

```javascript
// 创建 1 小时后的提醒
import { cron } from 'openclaw';

await cron.add({
  name: "会议提醒",
  schedule: {
    kind: "at",
    at: new Date(Date.now() + 60 * 60 * 1000).toISOString()
  },
  payload: {
    kind: "systemEvent",
    text: "会议将在 1 小时后开始！"
  },
  sessionTarget: "main"
});
```

### 循环任务 (every)

```bash
# 每隔 30 分钟检查一次
openclaw cron add \
  --name "定期检查" \
  --schedule "every 30m" \
  --message "检查完成" \
  --session isolated
```

### Cron 表达式

```bash
# 每天早上 9 点
openclaw cron add \
  --name "早安提醒" \
  --schedule "cron 0 9 * * *" \
  --message "早上好！新的一天开始了~" \
  --session isolated
```

常见 Cron 表达式：

| 表达式 | 说明 |
|--------|------|
| `0 9 * * *` | 每天 9:00 |
| `0 9 * * 1-5` | 工作日 9:00 |
| `0 9 * * 0,6` | 周末 9:00 |
| `*/30 * * * *` | 每 30 分钟 |
| `0 0 1 * *` | 每月 1 日 |

## 定时任务示例

### 天气提醒

```javascript
await cron.add({
  name: "天气提醒",
  schedule: {
    kind: "cron",
    expr: "0 7 * * *",
    tz: "Asia/Shanghai"
  },
  payload: {
    kind: "agentTurn",
    message: "查询上海今天的天气，发送给杰哥",
    timeoutSeconds: 60
  },
  sessionTarget: "isolated",
  delivery: {
    mode: "announce",
    channel: "telegram",
    to: "@your_telegram_id"
  }
});
```

### 定期备份

```javascript
await cron.add({
  name: "配置备份",
  schedule: {
    kind: "cron",
    expr: "0 2 * * *"
  },
  payload: {
    kind: "agentTurn",
    message: "执行备份脚本，保存配置到 /backup/"
  },
  sessionTarget: "isolated"
});
```

### 健康检查

```bash
# 每 5 分钟检查 Gateway 状态
openclaw cron add \
  --name "Gateway健康检查" \
  --schedule "every 5m" \
  --message "检查系统状态" \
  --session isolated
```

## 消息内容模板

### 添加上下文

```javascript
await cron.add({
  name: "会议提醒",
  schedule: {
    kind: "at",
    at: "2026-03-05T14:00:00Z"
  },
  payload: {
    kind: "agentTurn",
    message: "提醒杰哥：下午 2 点与张三的会议"
  },
  sessionTarget: "isolated",
  contextMessages: 5  // 添加最近 5 条消息作为上下文
});
```

### 传递参数

```javascript
await cron.add({
  name: "定时发送",
  schedule: {
    kind: "cron",
    expr: "0 10 * * *"
  },
  payload: {
    kind: "agentTurn",
    message: "发送日报给杰哥，汇总昨日数据"
  },
  sessionTarget: "isolated"
});
```

## 高级配置

### 禁用任务

```javascript
await cron.update("job-id", {
  enabled: false
});
```

### 任务失败重试

```json
{
  "schedule": {
    "kind": "cron",
    "expr": "0 9 * * *"
  },
  "retry": {
    "maxAttempts": 3,
    "delayMs": 60000
  }
}
```

### 任务依赖

```json
{
  "schedule": {
    "kind": "cron",
    "expr": "0 10 * * *"
  },
  "dependencies": ["backup-job-id"]

```

## 最佳实践

1. **使用有意义的名称**：便于识别和管理
2. **设置合理的超时**：避免任务卡住
3. **启用交付确认**：确保消息发送成功
4. **定期清理任务**：删除已完成的提醒
5. **时区设置**：确保 Cron 表达式时区正确

## 常见问题

### Q: 任务没有执行？

1. 检查任务是否启用
2. 查看任务运行历史
3. 检查日志输出

### Q: 时区不一致？

```javascript
schedule: {
  kind: "cron",
  expr: "0 9 * * *",
  tz: "Asia/Shanghai"  // 明确指定时区
}
```

### Q: 任务执行时间不准确？

- `every`：允许一定时间漂移（适合非关键任务）
- `cron`：精确执行（适合重要定时任务）
- `at`：精确执行一次性任务

### Q: 如何调试任务？

```bash
# 手动触发任务
openclaw cron run <jobId>

# 查看详细日志
openclaw logs --level debug | grep cron
```
